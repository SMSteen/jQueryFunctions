jQuery Methods - The Basics

I’m learning jQuery Methods, and now you can too! This document will address the basics and reveal some of the cool stuff you can do with jQuery methods. Point and click away. Happy learning.

Let’s start with Effects methods. These methods let you do simple things like hide or show webpage elements and add nifty animations.

.hide() and .show do exactly what you would think. They hide and show things.

.toggle() alternates between showing and hiding each time the method is called.

Click here to move on...

.slideUp also hides things but with a little built in effect. It transitions an element up until it disappears altogether.

.slideDown, as you may have guessed, does the opposite. It transitions things down until they are fully displayed.

.slideToggle does both by alternating between sliding things Up and Down. Select the button, then select it again, to see for yourself.

Click here to move on...

.fadeIn and .fadeOut also make things hide and display, using a fade effect.

Let’s move on to CSS methods. These methods let you access, or even change, CSS properties.

.addClass() adds a class to the specified element. That might come in handy if you've already set up some CSS styling for that class.

Lastly, lets take a look at some Events and Manipulation methods.

Every time you've clicked things on this webpage, you've triggered the .click() event. You clicked, and the webpage took a specified action. It's pretty much that simple.

Notice the textbox below the button just changed, as it has all along. Well that was accomplished by using either the .text() or .html() methods which manipulate the actual webpage. What's the difference? Click the button again to find out.

So what if you don't want to overwrite existing elements? That's where .prepend(), .append(), .before(), and .after() all come in handy. Whoa, that's a lot of manipulation methods. What do they all do? Press those buttons and find out!

Almost done. We're going to end with a quick look at the .val() and .data() methods. .val() is primarily used to get, or sometimes set, values in form fields.

.data() gets, or sets, arbitrary data for an element. Wait, isn't that what .val()does too? Well not quite. In our same example, using the .data() method, I can store the value you just keyed in, and use it later, even if you change that value.
